Documentation

The documentation is divided into parts:

  • Tool manual. A manual for how to interact with the compiler. This includes a discussion of where to place files and how to locate them.
  • Language documentation. A non-formal description of the core language. This includes both language constructs and their syntax, as well as a documentation of the global environment provided by the core.
  • Guide to extensions. An extension is a piece of software which provides funcitonality rather than producing output. This part of the documentation contains descriptions of some important extensions.
  • User's guide. An instructional text. This includes a tutorial, and goes further b discussing how the laguage constructs can be used by looking at several examples.
